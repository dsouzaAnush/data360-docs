# Limits & Guidelines

<Note>
As of October 14, 2025, Data Cloud has been rebranded to **Data 360**. During this transition, you may see references to Data Cloud in our application and documentation.
</Note>

This page provides a reference for the most important limits, quotas, and guidelines when developing with Data 360. Always check the [official Salesforce limits documentation](https://help.salesforce.com/s/articleView?id=sf.c360_a_limits_and_guidelines.htm&type=5) for the most current values.

| API | Limit | Per |
|-----|-------|-----|
| **Query API** | 100 concurrent queries | Org |

| **Ingestion API (Streaming)** | 10 requests/sec per data stream | Data Stream |
| **Ingestion API (Bulk)** | 10 concurrent jobs | Org |
| **Ingestion API payload** | 200 MB per bulk job | Job |
| **Ingestion API streaming payload** | 200 KB per request | Request |
| **Connect REST API** | Standard Salesforce API limits | Org |
| **Metadata API** | Standard Salesforce limits | Org |

## Performance Guidelines

<AccordionGroup>
  <Accordion title="Query Optimization">
    - Always include a `LIMIT` clause to prevent unbounded result sets
    - Use `WHERE` filters early to reduce data scanned
    - Avoid `SELECT *` — specify only the fields you need
    - Use `EXPLAIN` to analyze query execution plans
    - Partition large queries using time-based filters
  </Accordion>

  <Accordion title="Ingestion Optimization">
    - Use streaming ingestion for real-time use cases, bulk for large historical loads
    - Keep individual streaming payloads under 200 KB
    - Batch related records together to reduce API call overhead
    - Monitor ingestion latency and error rates
  </Accordion>

  <Accordion title="Activation Optimization">
    - Use incremental syncs instead of full refreshes where possible
    - Schedule activations during off-peak hours
    - Monitor match rates and optimize identifier mappings
    - Avoid activating the same segment to too many targets simultaneously
  </Accordion>
</AccordionGroup>
