# Data 360 Development Environments

Learn about the development environments that are available for Salesforce partners and customer developers.

Salesforce ISV Partners create apps for sale and distribution to Salesforce customers. If you're an ISV Partner, create a Partner Developer edition org from your Environment Hub, then log a case with [Partner Support](https://partners.salesforce.com) to enable Data 360 on it.

If you're a Salesforce Consulting Partner, you can get information about setting up a Data 360 development environment by contacting your Partner Account Manager.

Customer developers create apps in-house for their own Data 360 instance. Sandboxes are now generally available for creating and testing your apps for Data 360. Both Partial Copy and Full sandboxes are supported, and your CRM data is automatically replicated so it's immediately available for use with Data 360. See [Create a Data Cloud Sandbox](https://help.salesforce.com/s/articleView?id=data.c360_a_data_cloud_sandbox_create.htm).

## Sandbox Environments

With Data 360 sandbox support (GA), you can:

- **Test integrations** — Validate data streams, connectors, and API integrations in an isolated environment
- **Develop automations** — Build and test Data Cloud-triggered flows and invocable actions
- **Validate configurations** — Test identity resolution rulesets, calculated insights, and segments before promoting to production

### Deploying Changes

Once testing is complete, promote changes from sandbox to production using:

- **DevOps Data Kit** — Collects and sequences Data 360 configuration for deployment
- **Change Sets** — Standard Salesforce deployment mechanism
- **Salesforce CLI** — Command-line deployment for CI/CD workflows
- **Metadata API** — Programmatic deployment for automated pipelines

See [Deploy Data 360 Changes Across Orgs](https://help.salesforce.com/s/articleView?id=data.c360_a_data_cloud_sandbox_deploy.htm) for detailed deployment instructions.

## Alternative: Second Org

If you prefer not to use a sandbox, you can get access to a second org. To find out how to get access to a Data 360 test org, contact your Account team. The org used for testing is separate from the production org. This second org is another production org, and service usage is metered and results in billing charges the same way it is on the production org. See [Cost and Usage](https://help.salesforce.com/s/articleView?id=data.c360_a_data_usage_types.htm&type=5).
