# Data Streams API

The Data Streams API allows you to programmatically manage data stream configurations, including creating, updating, and monitoring data streams that flow into Data 360.

## Overview

Data streams define how data enters Data 360 from external sources. Each stream has:

- **Schema definition** - Fields, data types, and validation rules
- **Source configuration** - Where data originates
- **Processing rules** - How data is transformed and enriched

## Endpoints

### List Data Streams

#### GET /services/data/v64.0/ssot/data-streams

Retrieve all data streams configured in your org.

**Query Parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `status` | string | Filter by stream status: `ACTIVE`, `INACTIVE`, `ERROR` |

## Code Examples

<CodeGroup>
```bash cURL icon=terminal
curl -X GET "https://{instance}.salesforce.com/services/data/v64.0/ssot/data-streams?status=ACTIVE" \
  -H "Authorization: Bearer {access_token}"
```

```python get_stream_details.py icon=python lines {4-5,8-10}
import requests

url = f"https://{instance}.salesforce.com/services/data/v64.0/ssot/data-streams/WebEngagement_Stream"
headers = {"Authorization": f"Bearer {access_token}"}

response = requests.get(url, headers=headers)
stream = response.json()

print(f"Stream: {stream['displayName']}")
print(f"Records: {stream['metrics']['recordCount']:,}")
print(f"Error Rate: {stream['metrics']['errorRate']:.2%}")
```

## Data Stream Categories

| Category | Description | Examples |
|----------|-------------|----------|
| **Profile** | Customer identity and demographic data | Contacts, Accounts, Loyalty Members |
| **Engagement** | Interaction and behavioral data | Web events, Email opens, Purchases |
| **Other** | Reference and supporting data | Products, Locations, Campaigns |

## Stream Status Values

| Status | Description |
|--------|-------------|
| `ACTIVE` | Stream is accepting and processing data |
| `INACTIVE` | Stream is paused, not accepting data |
| `ERROR` | Stream has configuration or processing errors |
| `PENDING` | Stream is being set up or modified |

## Best Practices

<AccordionGroup>
  <Accordion title="Monitor Stream Health">
    - Regularly check error rates and ingestion statistics
    - Set up alerts for unusual patterns or failures
    - Review top errors to identify data quality issues
  </Accordion>

  <Accordion title="Schema Management">
    - Document all field definitions and purposes
    - Use descriptive display names for fields
    - Plan for schema evolution and versioning
  </Accordion>

  <Accordion title="Performance Optimization">
    - Create secondary indexes for frequently queried fields
    - Choose appropriate data types to minimize storage
    - Archive or purge old data as needed
  </Accordion>
</AccordionGroup>
