# Activations API

The Activations API enables you to programmatically manage activation targets, configure data exports, and monitor the delivery of segment data to external platforms.

## Overview

Activations in Data 360 allow you to:

- **Export segments** to advertising platforms (Google Ads, Meta, etc.)
- **Sync data** to marketing automation tools
- **Send profiles** to data warehouses and analytics platforms
- **Monitor delivery** status and troubleshoot issues

### Create Activation

#### POST /services/data/v64.0/ssot/activations

Create a new activation to sync a segment to a target.

**Request Body:**

```json Request {2-3,4-14,15-18}
{
  "segmentId": "SEG_003",
  "targetId": "ACT_001",
  "fieldMappings": [
    {
      "sourceField": "Email__c",
      "destinationField": "hashedEmail"
    },
    {
      "sourceField": "Phone__c",
      "destinationField": "hashedPhone"
    }
  ],
  "syncSchedule": {
    "frequency": "DAILY",
    "time": "06:00:00Z"
  },
  "options": {
    "includeNewMembers": true,
    "removeExitedMembers": true
  }
}
```

**Response:**

```json Response {2-3}
{
  "id": "ACTV_003",
  "status": "PENDING",
  "message": "Activation created. Initial sync will begin at scheduled time."
}
```

### Trigger Activation Sync

#### POST /services/data/v64.0/ssot/activations/{activationId}/sync

Manually trigger an immediate sync for an activation.

<ParamField path="activationId" type="string" required>
  The unique identifier of the activation
</ParamField>

**Request Body (Optional):**

```json
{
  "syncType": "FULL",
  "priority": "HIGH"
}
```

**Response:**

```json
{
  "activationId": "ACTV_001",
  "syncJobId": "SYNC_001",
  "status": "RUNNING",
  "estimatedCompletionTime": "2024-01-15T10:45:00Z"
}
```

### Pause/Resume Activation

#### PATCH /services/data/v64.0/ssot/activations/{activationId}

Update activation status (pause or resume).

<ParamField path="activationId" type="string" required>
  The unique identifier of the activation
</ParamField>

**Request Body:**

```json
{
  "status": "PAUSED"
}
```

**Response:**

```json

  "id": "ACTV_001",
  "status": "PAUSED",
  "message": "Activation paused. No further syncs will occur until resumed."
}
```

```python trigger_sync.py icon=python lines {4-8,9,11-13}
import requests

activation_id = "ACTV_001"
url = f"https://{instance}.salesforce.com/services/data/v64.0/ssot/activations/{activation_id}/sync"
headers = {
    "Authorization": f"Bearer {access_token}",
    "Content-Type": "application/json"
}
payload = {"syncType": "INCREMENTAL"}

response = requests.post(url, headers=headers, json=payload)
result = response.json()
print(f"Sync job {result['syncJobId']} started with status: {result['status']}")
```

## Sync Types

| Type | Description | Use Case |
|------|-------------|----------|
| `FULL` | Re-sync all segment members | Initial sync, data reconciliation |
| `INCREMENTAL` | Only sync changes since last sync | Regular scheduled syncs |
| `ADDITIONS_ONLY` | Only add new members | Suppression list use cases |
| `REMOVALS_ONLY` | Only remove exited members | Cleanup operations |

## Best Practices

<AccordionGroup>
  <Accordion title="Connection Security">
    - Use OAuth 2.0 for platform authentication
    - Rotate credentials regularly
    - Enable data encryption in transit
    - Review connected app permissions periodically
  </Accordion>

  <Accordion title="Sync Optimization">
    - Use incremental syncs for large segments
    - Schedule syncs during platform low-usage periods
    - Monitor match rates and optimize identifier mappings
    - Set up alerts for sync failures
  </Accordion>

  <Accordion title="Data Privacy">
    - Hash PII before sending to ad platforms
    - Review data sharing agreements
    - Implement consent-based activation filters
    - Maintain audit logs of data exports
  </Accordion>

  <Accordion title="Troubleshooting">
    - Check sync history for error patterns
    - Verify field mappings match platform requirements
    - Test with small segments before large-scale activation
    - Contact platform support for persistent match rate issues
  </Accordion>
</AccordionGroup>
