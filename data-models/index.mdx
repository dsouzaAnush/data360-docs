# Standard Data Model Objects (DMOs)

<Snippet file="/snippets/note-rebranding.mdx" />

Data model objects (DMOs) are groupings of data created from data streams, insights, and other sources. Find reference information about the standard DMOs found in Data 360.

-   [**Account DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-account-dmo.html)

    The Account DMO represents how a party wants to interact with your company.

-   [**Account Contact DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-account-contact-dmo.html)

    The Account Contact DMO represents an individual who has a role specific to an account.

-   [**Affiliation DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-affiliation-dmo.html)

    The Affiliation DMO represents affiliation or how to map Marketing Cloud Engagement business unit use case data.

-   [**Agent Service Presence DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-agent-service-presence-dmo.html)

    The Agent Service Presence DMO represents a user's real-time presence status.

-   [**Agent Work DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-agent-work-dmo.html)

    The Agent Work DMO represents a work assignment that's been routed to an agent.

-   [**Agent Work Skill DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-agent-work-skill-dmo.html)

    The Agent Work Skill DMO represents a skill that's used to route a work assignment to an agent.

-   [**Applicant DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-applicant-dmo.html)

    The Applicant DMO represents an individual or group that's requesting something from an organization, such as a loan or admission to a university.

-   [**Asset Service Level Objective DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-asset-service-level-objective-dmo.html)

    The Asset Service Level Objective (SLO) DMO represents SLO data that's transmitted over a network, such as a computer network or a telecommunications network. SLO data stores details, such as source and destination IP addresses, timestamps, and data types (for example, TCP and UDP).

-   [**Asset Service Level Objective Consequence DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-asset-service-level-objective-consequence-dmo.html)

    The Asset Service Level Objective (SLO) Consequence DMO represents network communications data that's sent over a network when the SLO isn't met. These are computer networks or telecommunications networks that include all data packets sent and received over a network, such as emails, voice calls, video streaming, web browsing, or any other type of data transfer.

-   [**Authorization Form DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-authorization-form-dmo.html)

    The Authorization Form DMO represents a set of terms and conditions, such as a privacy policy, a contract, or consent forms.

-   [**Authorization Form Consent DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-authorization-form-consent-dmo.html)

    The Authorization Form Consent DMO represents capturing where, when, and how a party gives consent for a form, a set of terms and conditions, or a privacy policy.

-   [**Authorization Form Data Use DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-authorization-form-data-use-dmo.html)

    The Authorization Form Data Use DMO represents the data uses consented to in an authorization form.

-   [**Authorization Form Text DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-authorization-form-text-dmo.html)

    The Authorization Form Text DMO represents an authorization form's text and language settings.

-   [**Benefit Action DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-benefit-action-dmo.html)

    The Benefit Action DMO represents actions triggered when a program benefit is assigned to a loyalty program member.

-   [**Benefit Type DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-benefit-type-dmo.html)

    The Benefit Type DMO represents the types of benefits of a loyalty program, such as healthcare, financial, and loyalty.

-   [**Brand DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-brand-dmo.html)

    The Brand DMO represents the product's brand, such as Northern Trail Outfitters.

-   [**Card Account DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-card-account-dmo.html)

    The Card Account DMO represents a financial tool offered by a bank as a type of loan, with a line of revolving credit that you can access via your card and your card's account.

-   [**Case DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-case.html)

    The Case DMO represents support cases based on recorded issues, such as a laptop connectivity problem.

-   [**Case Update DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-case-update-dmo.html)

    The Case Update DMO represents historical information about changes made to the associated case.

-   [**Communication Subscription DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-communication-subscription-dmo.html)

    The Communication Subscription DMO represents a customer's subscription preferences for a specific communication.

-   [**Communication Subscription Channel Type DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-communication-subscription-channel-type-dmo.html)

    The Communication Subscription Channel Type DMO represents the engagement channel where a customer can be contacted for a communication subscription.

-   [**Communication Subscription Consent DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-communication-subscription-consent-dmo.html)

    The Communication Subscription Consent DMO represents a customer's engagement or communication channel preferences.

-   [**Communication Subscription Timing DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-communication-subscription-timing-dmo.html)

    The Communication Subscription Timing DMO represents a customer's timing preferences for receiving a communication subscription.

-   [**Consent Action DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-consent-action-dmo.html)

    The Consent Action DMO represents what a user consents to regarding their data, such as data collection or web activity tracking.

-   [**Consent Status DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-consent-status-dmo.html)

    The Consent Status DMO represents the status of consent, such as opted in or opted out of data collection.

-   [**Contact Point Address DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-contact-point-address-dmo.html)

    The Contact Point Address DMO represents the mailing address of a party.

-   [**Contact Point App DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-contact-point-app-dmo.html)

-   [**Contact Point Consent DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-contact-point-consent-dmo.html)

    The Conversation Reason DMO represents the reason a conversation started. It contains aggregated metrics for excerpts, such as cancel order, update order, and check on order status.

-   [**Conversation Reason Category DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-conversation-reason-category-dmo.html)

    The Conversation Reason Category DMO represents a grouping of conversation reasons that have the same overall topic. It contains aggregated metrics for the associated conversation reasons, such as order management, payments, and account management.

-   [**Conversation Reason Report Definition DMO**](https://developer.salesforce.com/docs/data/data-cloud-dmo-mapping/guide/c360dm-conversation-reason-report-definition-dmo.html)
