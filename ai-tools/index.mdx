## MCP server vs. skills — what's the difference?

They solve different problems and are designed to be used together.

| | Data 360 MCP Server | Salesforce Skills |
|---|---|---|
| **What it is** | A running server that exposes Data 360 APIs as agent **tools** | Markdown playbooks (`SKILL.md`) that teach the agent **how** to do Data 360 tasks |
| **Gives the agent** | The ability to *call* Data 360 (query, ingest, configure) | The know‑how and step‑by‑step workflows to do it correctly |
| **Runtime** | Node.js HTTP service (run locally or on Heroku) | Static files installed into your agent's skills directory |
| **Auth** | Connects to your Salesforce org with OAuth / tokens | No auth (guides the agent; calls go through the MCP server or CLI) |
| **Best for** | Letting an agent execute real API calls against your org | Encoding correct, repeatable Data Cloud procedures |

<Tip>
Install **both** for the best experience: the skills tell the agent *what* to do and the MCP server lets it *do* it against your Data 360 org.
</Tip>
