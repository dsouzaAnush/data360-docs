# API Reference

Complete reference for the Salesforce Interactions Web SDK's public API.

## Core Methods

### SalesforceInteractions.init()

Initializes the SDK. Must be called before any other method.

```javascript
SalesforceInteractions.init({
  cookieDomain: 'yourdomain.com',  // Optional
  consent: consentPromise           // Required: Promise<Consent[]>
}).then(() => {
  // SDK ready
});
```

### SalesforceInteractions.reinit()

Reinitializes the SDK with updated configuration. Use after changing cookie domain or other settings post-init.

### SalesforceInteractions.sendEvent()

Sends an event to Data 360:

```javascript
SalesforceInteractions.sendEvent({
  interaction: { /* event data */ },
  user: { /* optional user data */ },
  account: { /* optional account data */ }
});
```

## Identity Methods

| Method | Signature | Description |
|--------|-----------|-------------|
| `getAnonymousId()` | `(): string` | Returns the anonymous ID from the first-party cookie |
| `setAnonymousId(id)` | `(id: string): void` | Manually sets the anonymous ID |
| `resetAnonymousId()` | `(): void` | Deletes the cookie and generates a new anonymous ID |
| `getCookieDomain()` | `(): string` | Returns the current cookie domain |
| `setCookieDomain(domain)` | `(domain: string): void` | Sets the cookie domain (before init) |
| `clearIdentity()` | `(): void` | Clears the named identity, keeps anonymous ID |

See [Identity Management](/sdks/web-sdk/identity) for usage details.

## Consent Methods

| Method | Signature | Description |
|--------|-----------|-------------|
| `updateConsents(consents)` | `(consents: Consent \| Consent[]): void` | Update consent status at runtime |
| `getConsents()` | `(): ConsentWithMetadata[]` | Read current consent state |

See [Consent API](/sdks/web-sdk/consent) for usage details.

## Sitemap

The Sitemap module extracts data during page navigation, shares capture logic across pages, and separates data capture from presentation:

```javascript
SalesforceInteractions.initSitemap({
  global: {
    onActionEvent: (actionEvent) => {
      // Global handler for all page navigation
      return actionEvent;
    }
  },
  pageTypeDefault: {
    name: 'default',
    interaction: {
      name: 'Default Page'
    }
  },
  pageTypes: [
    {
      name: 'product_detail',
      isMatch: () => window.location.pathname.startsWith('/products/'),
      interaction: {
        name: 'Product Viewed',
        catalogObject: {
          type: 'Product',
          id: () => document.querySelector('[data-product-id]')?.dataset.productId
        }
      }
    },
    {
      name: 'category',
      isMatch: () => window.location.pathname.startsWith('/category/'),
      interaction: {
        name: 'Category Viewed'
      }
    }
  ]
});
```

### Sitemap Configuration

| Property | Type | Description |
|----------|------|-------------|
| `global` | object | Global handlers applied to all pages |
| `pageTypeDefault` | object | Default interaction for unmatched pages |
| `pageTypes` | array | Page-specific configurations with `isMatch` predicates |

Each `pageType` can define:
- `name` — Page type identifier
- `isMatch` — Function returning `true` if the current page matches
- `interaction` — Event data to send when matched (can use functions for dynamic values)

## Debugging

Enable console logging for development:

```javascript
// Set log level
SalesforceInteractions.log.setLevel('debug');
// Options: 'none' (default), 'error', 'warn', 'info', 'debug'
```

The logger wraps `console.log` and is accessible at `SalesforceInteractions.log`.

### Custom Event Listeners

The SDK exposes `CustomEvents` for binding listeners to internal SDK events:

```javascript
document.addEventListener('sfInteractions:eventSent', (event) => {
  console.log('Event sent:', event.detail);
});

document.addEventListener('sfInteractions:consentChanged', (event) => {
  console.log('Consent changed:', event.detail);
});
```
