# Integrate Your Mobile Applications

> **Note:**
> As of October 14, 2025, Data Cloud has been rebranded to Data 360. During this transition, you may see references to Data Cloud in our application and documentation. While the name is new, the functionality and content remains unchanged.

Integrate your mobile applications with Data 360 by using the Data 360 Module for the Engagement Mobile SDK. This integration captures rich behavioral and demographic data from customers, along with app lifecycle and screen navigation events. By collecting these events, Data 360 creates a unified customer profile that enables hyper-personalized engagement across multiple Salesforce clouds, including Marketing, Commerce, and Service.

To use the Data 360 Module, create a Salesforce data source connector and copy your tenant-specific endpoint. Start by logging into Data 360 and navigating to Websites and Mobile Apps. Create an [app connector](https://help.salesforce.com/s/articleView?id=data.c360_a_web_mobile_app_connector.htm) to configure your data sources.

To initialize the Data 360 Module, you’ll need the `appSourceId` and tenant-specific endpoint from your web data connector.

The Engagement Mobile SDK currently supports two major versions: [Version 3.x](https://developer.salesforce.com/docs/data/data-cloud-engagement-mobile-sdk/guide/c360a-api-engagement-mobile-sdk-v3.html) and [Version 2.x](https://developer.salesforce.com/docs/data/data-cloud-engagement-mobile-sdk/guide/c360a-api-engagement-mobile-sdk-v2.html). To take advantage of the latest features and improvements, upgrade to Version 3.x.

Download the binaries for the Data 360 Module from the official repositories.

-   [Android SDK](https://github.com/salesforce-marketingcloud/mobile-sdk-cdp-android)
-   [iOS SDK](https://github.com/salesforce-marketingcloud/mobile-sdk-cdp-ios)
